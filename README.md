# DGH Healthcraft(无伤大雅：医疗工艺)

一个围绕 疾病-营养-用药 闭环设计的生存医疗扩展模组。

本文档用于约束玩法目标、系统边界与实现方向，作为后续开发与平衡调优基线。

## 1. 设计目标

- 提供比原版更高压力、更可管理的生存健康系统。
- 让玩家在感染风险、营养管理、药物资源之间做取舍。
- 强化医疗行为的操作感：主副手配合、风险药针、阶段性病程。
- 保持强配置化，支持整合包与服务器自行调参。

## 2. 核心玩法闭环

1. 玩家在环境与战斗中感染疾病。
2. 疾病影响行动、恢复或其他致病概率。
3. 玩家通过营养调节、睡眠与药物进行控制。
4. 高风险治疗行为会引入新的感染风险。
5. 周期循环，形成长期生存决策。

## 3. 使用机制

### 3.1 胶囊/口服剂

- 作为常规口服治疗手段。
- 服药存在冷却与延迟生效逻辑，避免瞬时滥用。

### 3.2 药针/注射液

- 主手：药针（普通/受污染）。
- 副手：注射液（阻断剂、利巴韦林等）。
- 注射会消耗耐久与物品使用次数。
- 受污染药针存在附加感染风险。

### 3.3 扫描类工具

- 营养扫描仪用于读取目标营养状态。
- 作为治疗前置决策工具，减少盲目用药。

## 4. 疾病系统

### 4.1 艾滋病

- 感染来源：受污染药针。
- 影响：提高其他疾病触发风险。
- 治疗：拉米夫定（低概率治愈）。

### 4.2 上呼吸道感染

- 典型触发：雨天行为与身体状态触发。
- 分期：轻型 -> 中型 -> 重型。
- 关键特性：轻中型可部分逆转，重型自愈困难。
- 药物应对：
  - 右美沙芬：治疗轻型，概率缓解中型。
  - 布洛芬：稳定中型与重型症状。
  - 利巴韦林注射液：快速干预中型与重型。

### 4.3 创伤后应激障碍

- 触发：受击与威胁环境累积。
- 影响：负面状态叠加，降低战斗与采集效率。
- 缓解：睡眠与镇静类药物。

### 4.4 脓毒症

- 感染来源：受污染药针。
- 影响：额外营养消耗与中毒风险。
- 治疗：可兼容原 dontgethurt 相关抗生素链路。

### 4.5 尸毒感染与尸变

- 感染来源：亡灵攻击与可配置实体池。
- 病程：初期 -> 中期 -> 尸变。
- 影响：阻断部分自愈机制并持续恶化。
- 应对：
  - 靶向剂：延缓恶化。
  - 阻断剂注射液：提供阶段性防护窗口。

## 5. 营养系统

### 5.1 营养维度

- 水分 Water
- 糖类 Saccharides
- 油脂 Fats and Oil
- 蛋白质 Protein
- 无机盐 Inorganic Salt
- 维生素 Vitamin
- 膳食纤维 Dietary Fiber

### 5.2 设计原则

- 每个维度均存在 过低惩罚 与 过高惩罚。
- 疾病概率与恢复效率受营养状态调制。
- 睡眠、运动、战斗等行为会动态改变营养值。

### 5.3 预期体验

- 玩家需要通过饮食结构而非单一食物维持健康。
- 营养系统不仅影响生存，也影响疾病链路。

## 7. 合成与资源策略

- 医疗物品应保持明显的等级分层：
  - 基础耗材可早期获取。
  - 高价值治疗物品需要更高成本或前置链。
- 同类物品尽量统一配方风格，便于记忆与平衡。
- 若调整配方，需同步检查：
  - 物品注册名与配方结果 ID 一致。
  - 是否影响瓶装/胶囊/注射液的产出节奏。

## 8. 配置化要求

至少支持以下参数可配置：

- 感染概率（各疾病独立）。
- 药物冷却、延迟、效果强度。
- 药针耐久与污染相关参数。
- 尸毒防护时长与病程推进强度。
- 关键营养阈值与惩罚强度。

## 9. 开发与维护约定

- 文案统一由语言文件维护，避免硬编码文本。
- Tooltip 风格统一为分段结构，便于玩家阅读。
- 新增药物时必须同步更新：
  - 物品注册
  - 配方 JSON
  - 模型与语言键
  - 说明文档

## 10. 后续迭代建议

- 增加病历或状态面板，提升反馈可见性。
- 引入更明确的早中后期医疗科技线。
- 为整合包提供预设配置模板（轻量/标准/硬核）。

## 11. 开发环境与运行

### 11.1 导入依赖

- 本项目为 Gradle 项目，依赖通过 build.gradle 与 Gradle 仓库自动管理。
- 首次拉取代码后，先在 IDE 中导入为 Gradle 工程并执行依赖同步。
- 若依赖异常，可执行以下命令强制刷新：
  - gradle --refresh-dependencies

### 11.2 IDE 导入建议

- IntelliJ IDEA：
  - 打开项目根目录。
  - 以 Gradle 项目导入并等待索引完成。
- VS Code：
  - 安装 Java 与 Gradle 相关扩展。
  - 打开工作区后等待 Gradle 自动识别与同步。

### 11.3 常用构建与运行命令

- 构建项目：
  - gradle --refresh-dependencies
  - gradle build
- 启动客户端开发环境：
  - gradle runClient

### 11.4 可选命令

- 清理构建缓存：
  - gradle clean
- 重新生成并下载依赖后构建：
  - gradle --refresh-dependencies
  - gradle build
